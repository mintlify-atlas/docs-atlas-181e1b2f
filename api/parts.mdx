---
title: Parts API
description: Complete API reference for managing automotive parts including CRUD operations and filtering
---

The Parts API provides endpoints to create, read, update, and delete automotive parts in the catalog. Parts can be filtered by name or application vehicle, and can be associated with multiple vehicles.

## Get All Parts

<api method="GET" endpoint="/api/v1/parts" />

Retrieves all parts in alphabetical order by name. Supports optional filtering by part name or application vehicle.

### Query Parameters

<ParamField query="name" type="string" optional>
  Filter parts by name (case-insensitive, partial match). Returns all parts whose name contains this keyword.
</ParamField>

<ParamField query="vehicle" type="string" optional>
  Filter parts by application vehicle (case-insensitive, partial match). Returns all parts whose applicationVehicle field contains this keyword.
</ParamField>

### Response

<ResponseField name="id" type="long">
  Unique identifier for the part (auto-generated)
</ResponseField>

<ResponseField name="name" type="string">
  Name of the part
</ResponseField>

<ResponseField name="applicationVehicle" type="string">
  Description of which vehicles this part applies to
</ResponseField>

<ResponseField name="netWeight" type="double">
  Net weight of the part in the configured unit
</ResponseField>

<ResponseField name="grossWeight" type="double">
  Gross weight of the part in the configured unit
</ResponseField>

<ResponseField name="vehicles" type="array">
  Array of vehicle objects associated with this part
  <Expandable title="Vehicle Object">
    <ResponseField name="id" type="long">
      Unique identifier for the vehicle
    </ResponseField>
    <ResponseField name="name" type="string">
      Name of the vehicle
    </ResponseField>
  </Expandable>
</ResponseField>

### Example Request

<CodeGroup>
```bash Get all parts
curl -X GET http://localhost:8080/api/v1/parts
```

```bash Filter by name
curl -X GET "http://localhost:8080/api/v1/parts?name=polia"
```

```bash Filter by vehicle
curl -X GET "http://localhost:8080/api/v1/parts?vehicle=fusca"
```
</CodeGroup>

### Example Response

```json 200 OK
[
  {
    "id": 1,
    "name": "Polia",
    "applicationVehicle": "Fusca",
    "netWeight": 9.87,
    "grossWeight": 10.98,
    "vehicles": [
      {
        "id": 1,
        "name": "Fusca 1300"
      }
    ]
  },
  {
    "id": 2,
    "name": "Vela de Ignição",
    "applicationVehicle": "Universal",
    "netWeight": 0.05,
    "grossWeight": 0.08,
    "vehicles": []
  }
]
```

```json 204 No Content
```

<ResponseExample>
```json 500 Internal Server Error
```
</ResponseExample>

## Create Part

<api method="POST" endpoint="/api/v1/parts" />

Creates a new part in the catalog.

### Request Body

<ParamField body="name" type="string" required>
  Name of the part
</ParamField>

<ParamField body="applicationVehicle" type="string" optional>
  Description of which vehicles this part applies to
</ParamField>

<ParamField body="netWeight" type="double" required>
  Net weight of the part. Must be less than or equal to grossWeight.
</ParamField>

<ParamField body="grossWeight" type="double" required>
  Gross weight of the part. Must be greater than or equal to netWeight.
</ParamField>

<Note>
  The `id` field is auto-generated and should not be included in the request body.
</Note>

### Response

<ResponseField name="id" type="long">
  Auto-generated unique identifier for the newly created part
</ResponseField>

<ResponseField name="name" type="string">
  Name of the part
</ResponseField>

<ResponseField name="applicationVehicle" type="string">
  Description of which vehicles this part applies to
</ResponseField>

<ResponseField name="netWeight" type="double">
  Net weight of the part
</ResponseField>

<ResponseField name="grossWeight" type="double">
  Gross weight of the part
</ResponseField>

<ResponseField name="vehicles" type="array">
  Empty array (no vehicles associated yet)
</ResponseField>

### Example Request

```bash
curl -X POST http://localhost:8080/api/v1/parts \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Polia",
    "applicationVehicle": "Fusca",
    "netWeight": 9.87,
    "grossWeight": 10.98
  }'
```

### Example Response

```json 201 Created
{
  "id": 1,
  "name": "Polia",
  "applicationVehicle": "Fusca",
  "netWeight": 9.87,
  "grossWeight": 10.98,
  "vehicles": []
}
```

```json 500 Internal Server Error
```

## Update Part

<api method="PUT" endpoint="/api/v1/parts/{id}" />

Updates an existing part's information.

### Path Parameters

<ParamField path="id" type="long" required>
  The unique identifier of the part to update
</ParamField>

### Request Body

<ParamField body="name" type="string" required>
  Updated name of the part
</ParamField>

<ParamField body="applicationVehicle" type="string" optional>
  Updated description of which vehicles this part applies to
</ParamField>

<ParamField body="netWeight" type="double" required>
  Updated net weight of the part
</ParamField>

<ParamField body="grossWeight" type="double" required>
  Updated gross weight of the part
</ParamField>

### Response

<ResponseField name="id" type="long">
  Unique identifier of the updated part
</ResponseField>

<ResponseField name="name" type="string">
  Updated name of the part
</ResponseField>

<ResponseField name="applicationVehicle" type="string">
  Updated application vehicle description
</ResponseField>

<ResponseField name="netWeight" type="double">
  Updated net weight
</ResponseField>

<ResponseField name="grossWeight" type="double">
  Updated gross weight
</ResponseField>

<ResponseField name="vehicles" type="array">
  Array of associated vehicles (unchanged by this operation)
</ResponseField>

### Example Request

```bash
curl -X PUT http://localhost:8080/api/v1/parts/1 \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Polia Atualizada",
    "applicationVehicle": "Fusca e Kombi",
    "netWeight": 10.50,
    "grossWeight": 11.75
  }'
```

### Example Response

```json 200 OK
{
  "id": 1,
  "name": "Polia Atualizada",
  "applicationVehicle": "Fusca e Kombi",
  "netWeight": 10.50,
  "grossWeight": 11.75,
  "vehicles": [
    {
      "id": 1,
      "name": "Fusca 1300"
    }
  ]
}
```

```json 404 Not Found
```

## Delete Part

<api method="DELETE" endpoint="/api/v1/parts/{id}" />

Deletes a specific part from the catalog.

### Path Parameters

<ParamField path="id" type="long" required>
  The unique identifier of the part to delete
</ParamField>

### Example Request

```bash
curl -X DELETE http://localhost:8080/api/v1/parts/1
```

### Example Response

```text 204 No Content
```

```text 500 Internal Server Error
```

<Warning>
  Deleting a part will also remove all relationships between that part and any associated vehicles.
</Warning>

## Delete All Parts

<api method="DELETE" endpoint="/api/v1/parts" />

Deletes all parts from the catalog.

### Example Request

```bash
curl -X DELETE http://localhost:8080/api/v1/parts
```

### Example Response

```text 204 No Content
```

```text 500 Internal Server Error
```

<Warning>
  This operation is destructive and cannot be undone. All parts and their vehicle relationships will be permanently deleted.
</Warning>

## Get Vehicles by Part

<api method="GET" endpoint="/api/v1/parts/{partId}/vehicles" />

Retrieves all vehicles associated with a specific part.

### Path Parameters

<ParamField path="partId" type="long" required>
  The unique identifier of the part
</ParamField>

### Response

<ResponseField name="id" type="long">
  Unique identifier of the vehicle
</ResponseField>

<ResponseField name="name" type="string">
  Name of the vehicle
</ResponseField>

### Example Request

```bash
curl -X GET http://localhost:8080/api/v1/parts/1/vehicles
```

### Example Response

```json 200 OK
[
  {
    "id": 1,
    "name": "Fusca 1300"
  },
  {
    "id": 2,
    "name": "Kombi 1.6"
  }
]
```

```json 404 Not Found
```

<Note>
  Returns an empty array `[]` if the part exists but has no associated vehicles.
</Note>
