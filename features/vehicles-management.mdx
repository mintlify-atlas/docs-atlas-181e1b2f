## Overview

The Vehicles Management feature allows users to maintain a catalog of vehicles that can be associated with automotive parts. This creates a many-to-many relationship where parts can be linked to multiple vehicles and vehicles can have multiple compatible parts.

## Vehicle Entity Model

The Vehicle entity represents an automotive vehicle in the system.

```java Vehicle.java
@Entity
@Table(name = "vehicle")
public class Vehicle {
    @Id
    @GeneratedValue(strategy = GenerationType.AUTO)
    private Long id;

    @Column(name = "name")
    private String name;

    @ManyToMany(mappedBy = "vehicles")
    @JsonIgnore
    private Set<Part> parts = new HashSet<>();

    public Vehicle() {}

    public Vehicle(String name) {
        this.name = name;
    }

    // Getters and setters...
}
```

### Entity Attributes

- **id**: Auto-generated unique identifier
- **name**: Vehicle name or model (required)
- **parts**: Set of related parts (many-to-many relationship, JSON ignored to prevent circular references)

<Note>
  The `@JsonIgnore` annotation on the parts collection prevents circular reference issues when serializing Vehicle objects to JSON.
</Note>

### Retrieve Vehicles

Retrieve vehicles from the catalog with optional name filtering.

<Tabs>
  <Tab title="Get All Vehicles">
    ```java VehicleController.java:37
    @GetMapping("/vehicles")
    @ApiOperation(value = "Exibe todos os veículos em ordem alfabética")
    @ResponseStatus(HttpStatus.OK)
    public ResponseEntity<List<Vehicle>> getAllVehicles(
        @RequestParam(required = false) String name
    ) {
        try {
            List<Vehicle> vehicles = new ArrayList<>();
            if (name == null) {
                vehicleRepository.findByOrderByNameAsc()
                    .forEach(vehicles::add);
            } else {
                vehicleRepository.findByNameContainingIgnoreCaseOrderByNameAsc(name)
                    .forEach(vehicles::add);
            }

            if (vehicles.isEmpty()) {
                return new ResponseEntity<>(HttpStatus.NO_CONTENT);
            }
            return new ResponseEntity<>(vehicles, HttpStatus.OK);
        } catch (Exception e) {
            return new ResponseEntity<>(null, HttpStatus.INTERNAL_SERVER_ERROR);
        }
    }
    ```

    **Endpoint**: `GET /api/v1/vehicles`

    **Query Parameters**:
    - `name` (optional): Filter vehicles by name (case-insensitive partial match)

    **Example**:
    - `GET /api/v1/vehicles` - Returns all vehicles
    - `GET /api/v1/vehicles?name=fusca` - Returns vehicles containing "fusca"
  </Tab>
  <Tab title="Get Vehicle by ID">
    ```java VehicleController.java:58
    @GetMapping("/vehicles/{id}")
    @ApiOperation(value = "Exibe um veículo específico")
    @ResponseStatus(HttpStatus.OK)
    public ResponseEntity<Vehicle> getVehicleById(@PathVariable("id") Long id) {
    ```

    **Endpoint**: `GET /api/v1/vehicles/{id}`

    **Path Parameters**:
    - `id`: Vehicle ID

    **Response**:
    - 200 OK with vehicle data
    - 404 Not Found if vehicle doesn't exist
  </Tab>
</Tabs>

### Update a Vehicle

Modify an existing vehicle's information.

```java VehicleController.java:82
@PutMapping("/vehicles/{id}")
@ApiOperation(value = "Atualiza um veículo")
@ResponseStatus(HttpStatus.OK)
public ResponseEntity<Vehicle> updateVehicle(
    @PathVariable("id") Long id,
    @RequestBody Vehicle vehicle
) {
    Optional<Vehicle> vehicleData = vehicleRepository.findById(id);
    if (vehicleData.isPresent()) {
        Vehicle _vehicle = vehicleData.get();
        _vehicle.setName(vehicle.getName());
        return new ResponseEntity<>(vehicleRepository.save(_vehicle), HttpStatus.OK);
    }
    return new ResponseEntity<>(HttpStatus.NOT_FOUND);
}
```

**Endpoint**: `PUT /api/v1/vehicles/{id}`

**Request Body**:
```json
{
    "name": "Volkswagen Fusca 1969"
}
```

**Response**:
- 200 OK with updated vehicle data
- 404 Not Found if vehicle doesn't exist

## Search and Filter

### Search by Name

The vehicle listing supports case-insensitive partial name matching.

**How it works**:

1. User enters search text in the vehicle name filter
2. System performs case-insensitive search
3. Returns all vehicles containing the search term
4. Results are sorted alphabetically by name

**Example**:
- Search for "fusca" returns: "Fusca", "Fusca 1969", "VW Fusca"
- Search for "VW" returns: "VW Fusca", "VW Gol", "VW Kombi"

### Alphabetical Sorting

All vehicle listings are automatically sorted alphabetically by name in ascending order, making it easy to browse and locate specific vehicles.

## Repository Methods

The VehicleRepository provides custom query methods for efficient data retrieval:

```java
public interface VehicleRepository extends JpaRepository<Vehicle, Long> {
    // Find all vehicles ordered by name ascending
    List<Vehicle> findByOrderByNameAsc();

    // Find vehicles by name containing keyword (case-insensitive)
    List<Vehicle> findByNameContainingIgnoreCaseOrderByNameAsc(String name);
}
```

These methods leverage Spring Data JPA's query derivation to automatically generate optimized SQL queries.

## Best Practices

## Use Cases

### Scenario 1: Adding a New Vehicle Line

When your catalog expands to include parts for a new vehicle:

1. Create the vehicle entry
2. Navigate to each compatible part
3. Link the new vehicle to existing parts
4. Or add new parts specific to this vehicle

### Scenario 2: Managing Vehicle Variants

For vehicles with multiple variants (different years, trims, etc.):

1. Create separate vehicle entries for each variant
2. Link shared parts to multiple vehicle variants
3. Add variant-specific parts as needed

### Scenario 3: Catalog Cleanup

To maintain a clean, organized catalog:

1. Use search to find duplicate or outdated vehicles
2. Review parts relationships before deletion
3. Merge or update vehicle entries as needed
4. Delete obsolete vehicles
